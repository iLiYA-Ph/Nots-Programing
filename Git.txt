 git init :
ساخت فایل .git 

 git status :
چک کردن وضعیت فایل ها

 git add <file> :
اضافه کردن فایل به stage 
مثال :
 git add index.html 
برای اضافه کردن همه ی فایل ها :
 git add -A
برای اضافه کردن فایل هایی که اول اسم انها یک عبارت خاص دارند : 
 مثلا : page : git add "page*" 
یا اخرشان با یه پسوند خاصی تمام میشود ... :
 git add "*.html"

برای حذف فایل از گیت و فایل سیستم :
 git rm <file>

برای ثبت یا کامیت فایل ها : 
 git commit -m "پیام مرتبط با فایل یا تایتل"

دیدن لاگ ها :
 git log

مشاهده تمام فایل ها :
 git ls-files

فایل های تغییر یافته توی هد رو با نسخه قبلیشون مقایسه میکنه :
 git diff HEAD

فایل های تغییر یافته استیج رو با نسخه قبلیشون مقایسه میکنه :
 git diff --staged

فایل رو از استیج خارج میکنه :
 git reset <file>

فایل مورد نظره ادیت شده که خارج از استیجه رو به نسخه قبلی که ثبتش کرده بود برمیگردونه :
 git checkout -- <file>

================ - B R A N C H - ==================

مشاهده برنچ های قابل استفاده :
 git branch

ساخت یک برنچ جدید :
 git branch <name>

حذف برنچ :
 git branch -d <name>

استفاده از برنچ و انتخاب کردن :
 git checkout <name>

مرج کردن برنچ ها باهم :
مثال : 
ما توی برنچ (مستر) هستیم و میخوایم اونو با برنچ (فیکس پیج) مرج کنیم :
 git merge fixPages

مرحله قدم به قدم ساخت برنچ و مارج کردن 
1 - یک برنچ میسازیم
2 - وارد آن برنچ میشیم
3 - فایل ها را در آن برنچ فیکس میکینم
4 - سپس فایل ها را کامیت میکنیم
5 - و بر میگردیم به برنچ اصلی و آن را با برنچ فیکس مرج میکنیم

============= C L O N E - P R O J E C T ===========

کلون کردن یک پروژه یا همون اپلود :
 git clone <link repository in github>

ریختن تمام فایل ها در .git : 
 git add .

اد کردن ریپازیتوری صفحه به پروژه لوکال
 git remote add origin <link repository in github>

برای اینکه اولین بار پروژه رو توی گیت هاب ثبت کنیم :
 git push -u origin <branch name>
و دفعات بعدی برای اپدیت اون پروژه در گیت هاب : 
 git push
و برای استخراج فایل از گیت هاب : 
 git pull

--------------
اپدیت کردن پروژه ثبت شده در گیت هاب اگر از -u استفاده نکرده باشیم :
 git push origin master
--------------
================= G I T - T A G =================

مشاهده لیست تمام تگ ها :
 git tag

زدن تگ به یک کامیت :
 git tag -a <name tag> <hash commit> -m "<message>"
مثال :
 git tag -a Vx ef86a216c -m "edit GIT"

مشاهده جزییات یک تگ :
 git show <name tag>

اضافه کردن یک تگ به اورجین یا همان فایل تویه صفحه گیت هاب :
 git push origin <name tag>
